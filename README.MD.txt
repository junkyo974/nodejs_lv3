1. userId 부분에서 검색을 위해 param 을 사용하였고 외 다른 수정 및 생성이 필요한 부분에서 body를 사용하였습니다.
Request를 사용하는 부분에대해 공부하는 과정에서 param 은 리소스의 식별자를 전송하는데 사용되며 주로 GET에 많이 사용, body의 경우 HTTP 요청 메세지의 본문에 데이터를 전송하는 방법으로 post,put 등에 많이 쓰인다고
보았으나 아직 정확한 이해를 위해 좀 더 공부하는 단계입니다.

2. get - 데이터의 읽기 작업, 단순한 조회 부분에서 사용
   post - 리소스를 생성, 데이터를 새로 생성하는 대 사용
   put - 리소스를 수정, 생성된 데이터를 읽어와 수정하는대 사용
   delete - 리소를 삭제, 생성된 데이터를 삭제하는대 사용

3. RESTful한 API를 설계하지 못 했습니다. 전반적으로 기능을 구현하는 부분에 초점이 맞춰져 적절한 메소드를 사용하거나 리소스를 식별하는 부분에 있어 명확하지 못 하는 등 아직 부족한 부분이 많아 그렇게 생각합니다.

4. 역활별로 Directory Structure를 구분 할 경우 관련된 코드를 모아 디렉토리를 구분함으로써 코드의 유지 보수성이 향상되고 코드간의 의존성을 줄일 수 있는 것 같습니다. 또한 다른 사람이 보았을 때 코드의 구조를 명확하게
표현하고 가독성을 높일 수 있는 것 같습니다.
